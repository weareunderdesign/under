# Contributing to the  under  Font Project

thank you for considering contributing to  under . it’s people like you that make open-source projects meaningful and impactful. following these guidelines will help make collaboration smoother and ensure your contributions make a difference.

by contributing to the  under  font project, you agree that all your work becomes part of the project  under  the [SIL Open Font License, Version 1.1](http://scripts.sil.org/OFL).


## types of contributions we’re looking for

- **reporting issues**: for example, bad kerning pairs, inconsistent glyphs, or rendering issues.
- **fixes and improvements**: refining specific aspects like kerning, glyphs, or adapting the display designs.


## working with the font

the  under  font is edited entirely in [glyphs](https://glyphsapp.com/). all contributions should be made using this software. when editing:

- ensure all glyphs are consistent across masters (e.g., path order, anchor placement, and components).
- test changes within glyphs to maintain compatibility and visual consistency.


## recommended workflow

1. open the font in glyphs.
2. make your edits while ensuring consistency across masters.
3. thoroughly test changes within glyphs and preview across different sizes and weights.
4. export the font and review it in real-world scenarios (e.g., on websites or apps).

before submitting changes, double-check your edits for accuracy and consistency.

## faq

**do i need special tools to contribute?**
you only need the glyphs app to edit and review the font files.

**how do i report issues?**
submit issues or suggestions through the project’s official communication channels.
